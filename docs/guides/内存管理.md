# 🧠 内存管理

> box-demo 全部组件的内存分配模型、防泄漏策略、压力测试与排查方法

---

## 1. 分配层级（优先级从高到低）

| 优先级 | 方式 | 适用场景 | 例 |
|--------|------|----------|-----|
| **1** | 栈变量 | < 2KB，生命周期明确 | `int32_t raw[256]` |
| **2** | 静态 buffer | 组件级全局状态 | `s_text[16384]` |
| **3** | PSRAM (`heap_caps_malloc`) | > 10KB，非 DMA | `s_marquee_raw` 150KB |
| **4** | 堆 (`malloc`) | 临时动态大小 | `chat_task` 的 HTTP 响应 |

> **铁律**：能用栈不用堆，能用静态不用动态，能用 PSRAM 不用 DRAM。

## 2. FreeRTOS 任务参数铁律

```c
// ✅ 正确：任务体负责释放 strdup 参数
void my_task(void *arg) {
    char *data = (char *)arg;
    // ... 使用 data ...
    free(data);  // ← 必须！
    vTaskDelete(NULL);
}

// ❌ 错误：永不释放
void my_task(void *arg) {
    char *data = (char *)arg;
    // ... 使用 data ...
    vTaskDelete(NULL);  // data 泄漏！
}
```

### 错误路径也要释放

```c
char *host_copy = strdup(host);
if (xTaskCreate(..., host_copy, ...) != pdPASS) {
    free(host_copy);  // ← xTaskCreate 失败时必须释放！
}
```

### 优雅退出优于强杀

```c
// ❌ 强杀：任务清理代码（socket close、free 等）不执行
vTaskDelete(s_test_task);

// ✅ 优雅退出：先设标志，等待 500ms 让任务自行退出
s_test_running = false;
vTaskDelay(pdMS_TO_TICKS(500));
if (s_test_task) vTaskDelete(s_test_task);
```

## 4. 逐组件内存模型

### 4.1 Chat 组件

```
s_text[16384]    静态 DRAM  对话历史（~100 轮）
s_reply[2048]    静态 DRAM  单次 AI 回复
s_audio          malloc      单次 TTS 音频（~50KB 峰值）
HTTP data        malloc      单次响应（~30KB 峰值）
```

**生命周期**：

```
chat_send() → strdup(text)  ──→ chat_task → free(send_text)  ✅
                           ──→ malloc(data) → free(data)       ✅
                           ──→ malloc(s_audio) → chat_free_audio()  ✅
                                                    ↓
                          main.cpp Phase 4: 保存文件 → chat_free_audio()
                          chat_text_clear(): free(s_audio)    ✅
```

**用后即清原则**：`s_audio` 在保存到 SD 卡后立即 `chat_free_audio()`，不保留到下一轮。

### 4.2 ASR 组件

```
s_text[2048]        静态 DRAM  识别结果
s_last_result[512]  静态 DRAM  单次结果
chunk[4096]         栈 (4KB)   流式上传缓冲（4KB 分块）
```

**生命周期**：

```
asr_upload() → strdup(path)  ──→ upload_task → free(path)    ✅
                            ──→ malloc(fsize) → free(data)    ✅
                            ──→ esp_http_client_init → cleanup ✅
```

> ⚠️ 已优化为流式上传（4KB chunk），避免全文件 malloc（~600KB 峰值）。

### 4.3 Record 组件

```
raw[256]         栈 (1024B)   I2S 读取缓冲
pcm[256]         栈 (512B)    PCM 转换输出
placeholder[44]  栈 (44B)     WAV 头占位
```

**全部栈分配，零堆分配**。录音循环中最重的操作是 `fwrite` 到 SD 卡。

### 4.4 Audio 组件

```
buf[512]         栈 (1024B)   WAV 播放缓冲
```

循环 `fopen → fread → I2S write → fclose → fopen`。单文件播放，无堆分配。

### 4.5 Display 组件

| 资源 | 分配 | 大小 | 生命周期 |
|------|------|------|----------|
| `s_backbuffer` | PSRAM Sprite | 320×240×2=150KB | 全局 |
| PNG 解码 buf | `heap_caps_malloc`(PSRAM) | ~200KB | 临时，每次 `heap_caps_free` |
| GIF 帧 Sprite ×28 | PSRAM | ~2.24MB | 进入 GIF 模式一次 |
| `s_marquee_raw` | `heap_caps_malloc`(PSRAM) | 150KB | 进入走马灯一次 |

> GIF 帧和走马灯数据在退出对应模式时释放（`free_gif_frames()` / `heap_caps_free(s_marquee_raw)`）。

### 4.6 Storage / SD Card 组件

```
s_card           全局指针   sdmmc_card_t（由 FATFS 管理）
SPIFFS mount      系统      内部缓存由 CONFIG_SPIFFS_CACHE 控制
```

SD 卡 `sd_card_init()` 只执行一次。`sd_card_speed_test()` 中 `malloc(64KB)` 用于测速数据块，用后即 `free`。

### 4.7 Network 组件

```
host_copy/url_copy   strdup   ~30B   ping/http task 参数
params               strdup   ~30B   tcp task 参数
JSON buf             calloc   ~1KB   WiFi 扫描结果
```

全部已在任务体中 `free()`。`network_test_stop()` 改为优雅退出。

## 5. SPIFFS GC 陷阱（历史记录）

> ⚠️ 录音已迁移至 SD 卡，以下陷阱不再触发。保留作为历史参考。

### GC_MAX_RUNS

```ini
CONFIG_SPIFFS_GC_MAX_RUNS=2   # 建议值，默认 10 过高
```

### 删建同名文件导致碎片化滚雪球

```
录音1: 600KB → remove() → 600KB 脏页
录音2: 写入 → GC 回收脏页 → 卡顿
录音3: 更多脏页 → 更严重 GC → 倒计时冻结
```

**解决方案**：录音迁到 SD 卡 ✅

## 6. 缓冲大小设计指南

| 缓冲 | 大小 | 推算方法 |
|------|------|----------|
| `s_text` (chat) | 16384 | 每轮 ~150 字节 × 100 轮 ≈ 15KB |
| `s_text` (asr) | 2048 | 每句 ~80 字节 × 20 句 ≈ 1.6KB |
| `s_reply` | 2048 | DeepSeek max_tokens=300 → ~1800 字节 |
| `s_audio` | ~50KB | TTS 语音 2-3 秒 |
| I2S DMA | ~4KB | 默认 4 desc × 1024 字节 |

## 7. 压力测试方法

### 7.1 单一操作循环

```cpp
// 在子功能中循环操作 50 次，观察堆变化
for (int i = 0; i < 50; i++) {
    exercise();     // 执行一次完整操作
    vTaskDelay(pdMS_TO_TICKS(100));
}
ESP_LOGI(TAG, "After 50 cycles: free_heap=%lu PSRAM=%zu",
         esp_get_free_heap_size(),
         heap_caps_get_free_size(MALLOC_CAP_SPIRAM));
```

### 7.2 功能切换循环

```cpp
// 反复进入/退出子功能 20 次
for (int i = 0; i < 20; i++) {
    enter_subapp();
    exit_subapp();
}
```

### 7.3 内存监控 overlay

运行时在屏幕右上角显示堆/PSRAM 剩余（由 `draw_mem_overlay()` 实现），实时观察内存变化。
