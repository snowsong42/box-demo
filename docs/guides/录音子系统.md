# 🎙️ 录音子系统

> I2S1 RX → MEMS 麦克风 → WAV 文件，独立于音频播放的 I2S0 TX

---

## 硬件

| 信号 | GPIO | 方向 |
|------|------|------|
| I2S1 BCLK | 7 | OUT |
| I2S1 WS | 15 | OUT |
| I2S1 DIN | 1 | IN |
| 功放静音 (SD) | 2 | OUT |

---

## I2S 配置（不可修改！）

```c
// record.cpp record_init()
.slot_cfg = I2S_STD_PHILIPS_SLOT_DEFAULT_CONFIG(
    I2S_DATA_BIT_WIDTH_32BIT,   // 必须 32-bit
    I2S_SLOT_MODE_STEREO),      // 必须立体声
.sample_rate = 22050
```

**为什么不能改成 16-bit mono？**

INMP441 MEMS 麦克风输出 24-bit I2S 数据，需要 64 BCLK/帧（左右各 32 BCLK）。改为 16-bit mono 后 ESP32 只生成 16 BCLK/帧 → 麦克风时序错乱 → 白噪声 + 倒计时飞跑。

**数据提取**：I2S 收到 32-bit 立体声帧，但麦克风实际只有左声道有效数据。取 `raw[i*2] >> 16` 得 24-bit 采样的高 16 位。

## 功放静音时序

```
录音开始: GPIO2→LOW (关功放) → i2s_channel_enable → vTaskDelay(80ms) → s_capturing=true
录音结束: i2s_channel_disable → GPIO2→HIGH (开功放)
```

必须先关功放再开 I2S，防止麦克风啸叫。80ms 等待 INMP441 偏置稳定。

---

## record_is_capturing 标志

在 `vTaskDelay(80ms)` 后置 `true`，用于 ASR 状态机的 Phase 1→Phase 2 过渡。

```c
// record.cpp
s_capturing = true;   // 80ms 后
// ...
s_capturing = false;  // cleanup 时
```

---

## 任务优先级

```c
xTaskCreate(record_task, "record", 4096, NULL, 5, &s_record_task);
//                                            ↑ 优先级 5
```

优先级 5 确保录音不被优先级 1 的音频播放任务饿死。不要降到优先级 1。

## WAV 文件格式

| 参数 | 值 |
|------|------|
| 采样率 | 22050 Hz |
| 位深 | 16-bit |
| 声道 | Mono |
| 字节速率 | 44100 bytes/s |

头写入流程：先写 44 字节零占位 → 流式写 PCM → 最后 `fseek(0)` 回写真实头。

时间计算：`elapsed_sec = total_data / 44100`（44100 = 22050×2）。
