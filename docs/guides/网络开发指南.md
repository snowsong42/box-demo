# ESP32-S3 网络开发指南

> **项目**：box-demo | **芯片**：ESP32-S3 | **框架**：ESP-IDF v6.0.1
> **组件**：`net_test` | **最后更新**：2026-06-26

---

## 1. 概述

box-demo 项目基于 `net_test` 组件实现了完整的网络功能：

- **SoftAP 网页配网**：手机连热点 → 浏览器选 WiFi → 输密码 → 保存 NVS → 自动连接
- **连通性测试**：TCP connect 模拟 Ping
- **HTTP GET**：`esp_http_client` 请求示例
- **TCP Client**：`lwip/sockets` 原始 Socket 通信
- **WiFi 状态面板**：SSID / IP / RSSI / MAC 实时显示

所有功能通过主菜单的 4 个独立入口（Ping / HTTP / TCP / WiFi Status）访问。

## 3. WiFi 初始化：非阻塞事件驱动

### 设计要点

WiFi 初始化**不应阻塞主线程**。正确的做法是：注册事件回调 → 启动 WiFi → 立即返回，连接结果通过事件通知。

```cpp
void wifi_init(void)
{
    // 1. 注册事件处理器
    esp_event_handler_instance_register(
        WIFI_EVENT, ESP_EVENT_ANY_ID, &wifi_event_handler, NULL, NULL);
    esp_event_handler_instance_register(
        IP_EVENT, IP_EVENT_STA_GOT_IP, &wifi_event_handler, NULL, NULL);

    // 2. 从 NVS 读取已保存的凭据
    nvs_open("wifi_creds", NVS_READONLY, &nvs);
    nvs_get_str(nvs, "ssid", ssid, &len);
    nvs_get_str(nvs, "pass", pass, &len);
    nvs_close(nvs);

    // 3. 设置并启动（立即返回）
    esp_wifi_set_mode(WIFI_MODE_STA);
    esp_wifi_set_config(WIFI_IF_STA, &wifi_cfg);
    esp_wifi_start();

    // ⚠️ 不在此等待！事件回调中设置 s_connected 标志
}
```

### 事件回调关键逻辑

```cpp
static void wifi_event_handler(void *arg, esp_event_base_t event_base,
                               int32_t event_id, void *event_data)
{
    if (event_id == WIFI_EVENT_STA_START) {
        esp_wifi_connect();                       // 自动连接
    } else if (event_id == WIFI_EVENT_STA_DISCONNECTED) {
        if (retry < MAX_RETRY) esp_wifi_connect(); // 重试
        else s_connected = false;                  // 放弃
    } else if (event_id == IP_EVENT_STA_GOT_IP) {
        s_connected = true;                        // 连接成功
    }
}
```

> **教训**：切勿在 `wifi_init()` 中调用 `xEventGroupWaitBits(..., 10000ms)`，这会导致开机黑屏 10 秒。

## 4. SoftAP 网页配网

### 完整流程

```
ESP32 端                          手机端
────────                          ──────
启动 AP "box-demo" (密码 12345678)
启动 HTTP Server (:80)
                                 连接 WiFi "box-demo"
                                 浏览器打开 192.168.4.1
                                 点 [Scan] → 显示附近 WiFi 列表
                                 选择 WiFi + 输入密码 → [Connect]
收到 POST /connect
解析 JSON {ssid, password}
保存到 NVS
连接目标 WiFi ─────────────────→ 连接成功
停止 HTTP Server (设标志，主循环清理)
```

### HTTP Server 关键代码

```cpp
void wifi_prov_web_start(void)
{
    s_prov_done = false;                          // 重置完成标志
    esp_wifi_set_mode(WIFI_MODE_APSTA);           // AP + STA 双模

    wifi_config_t ap_cfg;
    memset(&ap_cfg, 0, sizeof(ap_cfg));           // ⚠️ 必须全零初始化
    strncpy((char *)ap_cfg.ap.ssid, "box-demo", 32);
    strncpy((char *)ap_cfg.ap.password, "12345678", 64);
    ap_cfg.ap.authmode = WIFI_AUTH_WPA_WPA2_PSK;  // ⚠️ 不要用 OPEN

    // 启动 HTTP Server
    httpd_start(&s_prov_server, &http_cfg);
    // 注册 3 个端点：GET /, GET /scan, POST /connect
}
```

### POST /connect 处理要点

```cpp
static esp_err_t prov_connect_handler(httpd_req_t *req)
{
    // 1. 读取请求体
    // 2. 手动 JSON 解析（纯 C，避免引入 cJSON 依赖）
    // 3. 保存到 NVS
    nvs_set_str(nvs, "ssid", ssid);
    nvs_set_str(nvs, "pass", pass);
    nvs_commit(nvs);
    // 4. 连接目标 WiFi
    esp_wifi_set_config(WIFI_IF_STA, &wifi_cfg);
    esp_wifi_connect();
    // 5. ⚠️ 只设标志，不要在此调用 httpd_stop()！
    s_prov_done = true;
}
```

> **教训**：在 HTTP handler 内调用 `httpd_stop()` 会导致死锁。handler 运行在 HTTP Server 的任务中，`httpd_stop()` 会等待该任务结束 → 自己等自己。正确做法是设 `s_prov_done = true`，由主循环检测并调用 `wifi_prov_web_stop()`。

### WiFi 扫描去重

```cpp
// 扫描结果按 SSID 去重，同名保留 RSSI 最高者
for (int i = 0; i < ap_count; i++) {
    bool dup = false;
    for (int j = 0; j < unique_count; j++) {
        if (strcmp(ap[i].ssid, ap[j].ssid) == 0) {
            if (ap[i].rssi > ap[j].rssi) ap[j] = ap[i];  // 保留强信号
            dup = true; break;
        }
    }
    if (!dup && ap[i].ssid[0]) ap[unique_count++] = ap[i];
}
```

## 5. 网络测试实现

### 5.1 Ping（连通性测试）

用 TCP connect 替代 ICMP Ping。ESP-IDF v6.0.1 的 `ping/ping_sock.h` API 不稳定，TCP connect 更可靠且足以验证连通性。

```cpp
for (int i = 0; i < 4 && s_test_running; i++) {
    int64_t start = esp_timer_get_time();
    // DNS 解析 + Socket + Connect
    getaddrinfo(host, "80", &hints, &res);
    int sock = socket(...);
    int ret = connect(sock, ...);
    int64_t elapsed = esp_timer_get_time() - start;
    // 输出 "OK 15ms" 或 "FAIL 2000ms"
    close(sock);
}
```

### 5.2 HTTP GET

```cpp
esp_http_client_config_t cfg = {};
cfg.url = "http://httpbin.org/get";  // ⚠️ 用纯 HTTP，避免 HTTPS 重定向
cfg.timeout_ms = 5000;
cfg.disable_auto_redirect = true;     // ⚠️ 禁用重定向
esp_http_client_handle_t client = esp_http_client_init(&cfg);
esp_http_client_perform(client);
```

> **教训**：`http://baidu.com` 会 301 重定向到 `https://www.baidu.com`，触发 TLS 证书错误 `ESP_ERR_MBEDTLS_SSL_SETUP_FAILED`。解决：用纯 HTTP 端点 + 禁重定向。

### 5.3 TCP Client

```cpp
// 发送标准 HTTP 请求（而非原始文本），获得可读响应
char send_data[128];
snprintf(send_data, sizeof(send_data),
         "GET / HTTP/1.0\r\nHost: %s\r\nConnection: close\r\n\r\n", host);
send(sock, send_data, strlen(send_data), 0);
```

> **教训**：向 web 服务器 80 端口发原始文本会收到 `400 Bad Request`。发送标准 HTTP 请求可获 `200 OK`。

## 6. 常见问题与解决

### 6.1 组件名冲突

| 问题 | 组件名 `network` 导致 ESP-IDF 构建系统报错 |
|------|-------------------------------------------|
| 根因 | `network` 是 ESP-IDF 内部保留名 |
| 解决 | 重命名为 `net_test` |

### 6.2 结构体字段初始化

| 问题 | `-Werror=missing-field-initializers`，大量编译错误 |
|------|---------------------------------------------------|
| 根因 | ESP-IDF v6.0.1 的 `wifi_ap_config_t`、`wifi_scan_config_t`、`esp_http_client_config_t` 等结构体新增字段，C99 指定初始化器 `{.field = val}` 未覆盖全字段 |
| 解决 | 放弃指定初始化器，改用 `memset(&s, 0, sizeof(s))` + 逐字段赋值 |

```cpp
// ❌ 错误：指定初始化器不全
wifi_scan_config_t scan_cfg = { .scan_type = WIFI_SCAN_TYPE_ACTIVE };

// ✅ 正确：memset + 赋值
wifi_scan_config_t scan_cfg;
memset(&scan_cfg, 0, sizeof(scan_cfg));
scan_cfg.scan_type = WIFI_SCAN_TYPE_ACTIVE;
```

### 6.3 SPIRAM PSRAM 未使能导致 malloc 失败

| 问题 | 启动时 SPIFFS mount 或大 buffer 分配失败 |
|------|-----------------------------------------|
| 根因 | sdkconfig 中未开启 PSRAM |
| 解决 | `menuconfig` → `Component config` → `ESP PSRAM` → 开启 |

### 6.4 Task 自删后悬空指针

| 问题 | 退出网络测试页后设备重启 |
|------|------------------------|
| 根因 | 测试 task 调用 `vTaskDelete(NULL)` 自删后，`s_test_task` 仍指向已删除 task。`network_test_stop()` 再次 `vTaskDelete(s_test_task)` 导致 crash |
| 解决 | 所有 task 自删前设置 `s_test_task = NULL` |

```cpp
// ❌ 错误
vTaskDelete(NULL);

// ✅ 正确
s_test_task = NULL;
vTaskDelete(NULL);
```

### 6.5 TLS 证书错误

| 问题 | `ESP_ERR_MBEDTLS_SSL_SETUP_FAILED` |
|------|-----------------------------------|
| 根因 | URL 被 301/302 重定向到 HTTPS，但未配置 TLS 证书 |
| 解决 | 使用纯 HTTP 端点 + `cfg.disable_auto_redirect = true` |

### 6.6 AP 开放网络兼容性

| 问题 | 手机要求输入 8 位密码才能连接 ESP32 热点 |
|------|---------------------------------------|
| 根因 | `WIFI_AUTH_OPEN` 在某些 Android/iOS 版本上不兼容 |
| 解决 | 改用 `WIFI_AUTH_WPA_WPA2_PSK` + 简单密码 `12345678` |

### 6.7 HTTP Server handler 内死锁

| 问题 | 配网完成后 BACK 按钮无响应 |
|------|--------------------------|
| 根因 | handler 内调用 `httpd_stop()` 等待自身任务结束 |
| 解决 | handler 只设 `s_prov_done = true`，主循环检测后异步清理 |

### 6.8 WiFi 扫描结果重复

| 问题 | 下拉框出现多个同名 WiFi |
|------|------------------------|
| 根因 | 同一 SSID 在多个信道广播 |
| 解决 | 按 SSID 去重，同名保留 RSSI 最高者 |
