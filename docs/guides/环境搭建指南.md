| 目录 | 安装方式 | 包含 |

```json
{

  "cmake.ignoreCMakeListsChanges": true,
  "cmake.useCMakePresets": "never",
  "idf.extensionActivationMode": "always",
  "idf.gitPathWin": "D:\\coding\\Git\\cmd\\git.exe",
  "idf.flashType": "UART",
  "idf.customExtraVars": {
    "IDF_PATH": "D:\\EE\\ESP-IDF\\.espressif\\v6.0.1\\esp-idf",
    "IDF_TOOLS_PATH": "C:\\Espressif\\tools",
    "IDF_PYTHON_ENV_PATH": "C:\\Espressif\\tools\\python\\v6.0.1\\venv"
  },
  "idf.customExtraPaths": [
    "C:\\Espressif\\tools\\xtensa-esp-elf\\esp-15.2.0_20251204\\xtensa-esp-elf\\bin",

```

| 配置项 | 作用 |
|--------|------|
| `cmake.configureOnOpen: false` | 禁止 CMake Tools 自动配置（ESP-IDF 项目由 `idf.py` 管理） |
| `idf.customExtraVars` | ESP-IDF 扩展 v2.x 的标准配置方式 |
| `idf.customExtraPaths` | 补充工具链和 Python 到 PATH |
| `idf.gitPathWin` | Git 路径（扩展验证 setup 时需要） |

## 3. 构建与烧录

### 3.1 快捷操作

| 操作 | VS Code 快捷键 | 说明 |
|------|---------------|------|
| 编译 | `Ctrl+Shift+B` | 调用 `idf.py build` |
| 烧录 | ESP-IDF 扩展按钮 | 点击底部状态栏火焰图标 |
| 监听 | ESP-IDF 扩展按钮 | 点击底部状态栏插头图标 |
| 清理 | `Ctrl+Shift+P` → `Tasks: Run Task` → `ESP-IDF: Clean` | 调用 `idf.py fullclean` |

### 3.2 命令行构建

```powershell
# 激活环境（如果使用 PowerShell 终端）
. "D:\EE\ESP-IDF\.espressif\v6.0.1\esp-idf\export.ps1"
idf.py build
idf.py -p COM8 flash
idf.py -p COM8 monitor
```

### 3.3 烧录分区布局

| 分区 | 地址 | 大小 | 内容 |
|------|------|------|------|
| bootloader | 0x00000000 | ~24KB | 引导程序 |
| partition-table | 0x00008000 | 4KB | 分区表 |
| factory (app) | 0x00010000 | 2MB | box-demo 主程序 |
| storage (SPIFFS) | 0x00210000 | 4MB | 图片/音频资源 |

## 4. `sdkconfig` 关键配置

以下 5 项必须正确，否则构建或运行会失败：

```ini
CONFIG_PARTITION_TABLE_CUSTOM=y
CONFIG_PARTITION_TABLE_CUSTOM_FILENAME="partitions.csv"
CONFIG_ESPTOOLPY_FLASHSIZE_16MB=y
CONFIG_SPIRAM=y
CONFIG_SPIRAM_MODE_QUAD=y
CONFIG_FREERTOS_HZ=1000
```

| 配置 | 说明 |
|------|------|
| `PARTITION_TABLE_CUSTOM` | 使用项目的 `partitions.csv`（含 SPIFFS storage 分区） |
| `FLASHSIZE_16MB` | HW-678 开发板 Flash 容量 |
| `SPIRAM` + `SPIRAM_MODE_QUAD` | 启用 PSRAM（LovyanGFX 后缓冲 + GIF 帧需要） |
| `FREERTOS_HZ=1000` | 1ms tick，项目所有 `vTaskDelay(pdMS_TO_TICKS(10))` 依赖此值 |

一键检查：
```powershell
Select-String -Path sdkconfig -Pattern 'PARTITION_TABLE_CUSTOM|FLASHSIZE_16MB|SPIRAM_MODE_QUAD|FREERTOS_HZ'
```

## 5. 常见错误速查

| 错误信息 | 原因 | 修复 |
|----------|------|------|
| `Current ESP-IDF setup is not found` | 扩展未找到 ESP-IDF 配置 | 检查 `.vscode/settings.json` 中的 `idf.customExtraVars`；执行 `ESP-IDF: Select Current ESP-IDF Version` |
| `export.bat` 激活失败（Python 版本不匹配） | 系统 PATH 有 Python 3.11，工具为 3.12 安装 | 不使用 `export.bat`，改用 `idf-exe` 封装器（`tasks.json` 已配置） |
| `No module named 'click'` | `idf.py.exe` 找不到 Python venv | 确保 `PATH` 包含 `venv\Scripts` 和 `IDF_PYTHON_ENV_PATH` 正确 |
| `cmake must be available on the PATH` | `idf.py` 找不到 cmake | 确保 `PATH` 包含 `cmake\4.0.3\bin` |
| `TypeError: expected string ... got 'NoneType'` (`ESP_IDF_VERSION`) | `idf_component_manager` 检查环境变量失败 | 设置 `ESP_IDF_VERSION=6.0.1` |
| `CMake Error: include could not find /tools/cmake/project.cmake` | CMake Tools 插件误配置项目 | 设置 `cmake.configureOnOpen: false`，ESP-IDF 项目不通过 CMake Tools 构建 |
| `PSRAM chip is not connected` | PSRAM 模式配置错误 | 检查 `CONFIG_SPIRAM_MODE_QUAD=y` |
| `Partitions ... does not fit in configured flash size 2MB` | Flash 大小默认 2MB | 设置 `CONFIG_ESPTOOLPY_FLASHSIZE_16MB=y` |

## 6. 项目组件依赖

```
main
├── REQUIRES: LovyanGFX, buttons, display, storage, audio
│
components/
├── buttons    → REQUIRES: esp_driver_gpio
├── display    → REQUIRES: LovyanGFX
├── storage    → REQUIRES: spiffs
├── audio      → REQUIRES: esp_driver_i2s
└── LovyanGFX  → (外部组件)
```

> ⚠️ ESP-IDF v6.0.1 将旧版单体 `driver` 拆分为 `esp_driver_gpio`、`esp_driver_i2s` 等子组件，需显式声明精确依赖。
