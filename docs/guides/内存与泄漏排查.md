# 🔍 内存与泄漏排查

> 本项目已发现并修复的内存泄漏清单及排查方法

---

## 已修复的泄漏

### 1. asr.cpp — upload_task 的 strdup 泄漏

```c
// asr_upload() 中
char *path_copy = strdup(filepath);
xTaskCreate(upload_task, ..., path_copy, ...);

// upload_task() 的 done: 标签处必须释放
done:
    free((void*)path);   // ← 此前缺失！
```

每次 ASR 上传泄漏 ~30 字节。

### 2. network.cpp — ping_test_task 的 strdup 泄漏

```c
// ping_start() 中
char *host_copy = strdup(host);
xTaskCreate(ping_test_task, ..., host_copy, ...);
// ping_test_task 结束前必须 free(host)
```

同样模式在 `http_test_task`（`free(url)`）和 `tcp_test_task`（`free(params)`，此项原本正确）。

### 3. network_test_stop — vTaskDelete 强杀

```c
// 错误写法
void network_test_stop(void) {
    if (s_test_task) {
        vTaskDelete(s_test_task);  // ← 强杀！不执行任务清理代码
    }
}
```

被强杀时，`http_test_task` 的 `esp_http_client_cleanup` 未执行，socket 未 `close`。

**修复**：先设标志 `s_test_running = false`，等待最多 500ms 让任务自行退出，最后手段才 `vTaskDelete`。

## 错误路径泄漏

各 `xTaskCreate` 调用失败时，`strdup` 产物未释放：

```c
char *host_copy = strdup(host);
if (xTaskCreate(..., host_copy, ...) != pdPASS) {
    // 缺少 free(host_copy)！
}
```

参考 `asr.cpp` 中正确处理：
```c
if (xTaskCreate(...) != pdPASS) {
    free(path_copy);
}
```

## SPIFFS GC 陷阱

### GC_MAX_RUNS

```ini
CONFIG_SPIFFS_GC_MAX_RUNS=2   # 建议值，默认 10 过高
```

### esp_spiffs_gc(SIZE_MAX) 导致看门狗复位

```c
// 危险操作！在 main 循环中阻塞数秒
esp_spiffs_gc("storage", SIZE_MAX);  // ← SIZE_MAX = GC 全部脏页，擦除 Flash 数秒
```

主循环阻塞 → idle 任务无法运行 → 任务看门狗超时 → 系统复位。**不要在任务上下文中调用 `SIZE_MAX` GC**。

### 删建同名文件导致碎片化滚雪球

```
录音1: 600KB → remove() → 600KB 脏页
录音2: 写入 → GC 回收脏页 → 卡顿
录音3: 更多脏页 → 更严重 GC → 倒计时冻结
```

**解决方案**：录音迁到 SD 卡。
